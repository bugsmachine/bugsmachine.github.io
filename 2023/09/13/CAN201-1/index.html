CAN201 CPT205 Introduction to Networking 1: protocol, layer and model…

Protocols define format, order of messages sent and received among network entities; and actions taken on message transmission, receipt…

A “layer” typically refers to a specific level or abstraction within a networking protocol stack, such as physical layer, network layer, application layer…

Network model often refers to a conceptual or architectural framework that helps in understanding and designing computer networks. Usually, there are two primary models OSI and TCP/IP:

  • OSI model (Open Systems Interconnection Model):
    OSI model consists of seven layers, each response for specific tasks within network communication.
  • TCO/IP model (Transmission Control protocol/Internet Protocol Model):
    This model is more commonly used in practice and consists 4 layers:
    • Application Layer (corresponds to OSI Layers 5, 6, and 7)
    • Transport Layer (corresponds to OSI Layer 4)
    • Internet Layer (corresponds to OSI Layer 3)
    • Link Layer (corresponds to OSI Layers 1 and 2)

Network Edge and Core

In computer networking, the network can be broadly divided into two key parts: the network edge and the network core.

  • Network Edge = Host
    Host includes Clients and Servers… Using wired or wireless communication links to access network.
  • Network Core:
    • Interconnected Routers
    • Network of Network

Different ways to access network

1. Dial-up Internet Access (PSTN):

Dial-up Internet Access is a form of internet access using facilities of public switches telephone network to connect to ISP (Internet Service Provider) by dialing a phone number on a conventional telephone line.

Important Equipment: Modems (Modulator and demodulator):
Decode: Audio signal -> Data -> send to router or pc…
Encoder: Encode data -> Audio signal -> send to other Modem

Important: Bandwidth: 56 Kbps ~ 10 min for a mp3 music

2. Digital subscriber line (DSL):

Also based on telephone line:
If DATA, goes to internet…
If VOICE, goes to telephone line…

Important: Bandwidth:

  • Upstream rate < 2.5 Mbps (typically < 1Mbps)
  • Downstream rate < 24 Mbps (typically < 10Mbps)

3. Cable network (TV net Based):

Key Technology of TV based cable network is Frequency division multiplexing (FDM):
Different channels transmitted in different frequency bands via hybrid fiber coax….

Important: Bandwidth: 30Mbps downstream and 2Mbps upstream

4. Enterprise access networks (Ethernet):

Normally used in institutions such as companies, universities, etc. Extremely high transmission rates.
End systems typically connect to Ethernet switch…

5. Wireless access network:

Shared wireless access network connects end system to router…

Different Medium

1. Guided media: Signal propagate in solid.

  • Twisted Pair (TP): Two insulated copper wires. (Figure 2)

    • Category 5: 100 Mbps, 1 Gbps Ethernet
    • Category 6: 10Gbps

  • Coaxial cable: Two concentric copper conductors. (Figure 3)

    • Bidirectional
    • Broadband -> multiple channels on cable (suitable for FDM)

  • Fiber optic cable: Glass fiber carrying light pules, each pile indicates a bit. (Figure 4)

    • High speed: point-to-point transmission 10-100 Gbs…
    • Low error rate: repeaters spaced far apart and immune to electromagnetic noise

2. Unguided media: Signal propagate freely.

  • Radio
    • Signal carried in electromagnetic spectrum
    • Essay to be affected by environment, such as interference by reflection, obstruction by objects and electromagnetic noise. !!Interference!!

How to link with others??

1. Package Switching:

Break applications-layer messages into small packages. Packages are forwarded form routers to routers via links.
Forwarding is one of the network-core function, which move packages from router’s input to appropriate router output. While the other function is Routing, which used a routing algorithm to determine source destination route taken by packets.

Queuing and loss:

  • If arrival rate (in bits) to link exceeds transmission rate of link for a period of time:
    • packets will queue, wait to be transmitted on link
    • packets can be dropped (lost) if memory (buffer) fills up
