Introduction to My APP: Academic Calculator

Introduction to My APP: Academic Calculator

This desktop application’s GUI is written by JavaFX and it’s back-end API is written in Django. Also, Mysql is employed to have a bigger storage database.

Front-end Part

There are 6 stages (windows) in total constituted this app’s GUI including log in stage, sign in stage, reset phone number stage, forget password stage, main (calculation) stage, and user profile stage.

Log In Interface

Back-end Support about Log In. Click Here

In this interface, user can log in, sign in, and reset password (shows in figure 1).

It achieve the functionality to show and hide the password via two container fields (shows in figure 2 and 3).

Meanwhile, It will popup alert when user enter the wrong/empty password, and loss connection with API (shows in figure 4,5,6).

Especially, if user enter a wrong password for a existed username, they can choose to reset password (open reset password stage). If no such user, the alert will lead user to sign in stage.

Sign In Interface

Back-end Support about Sign In. Click Here

User need to enter their username, password, and phone number (optional). Figure 7, 8, 9 shows corresponding warning and alert will show if user enter empty username/password or enter two wrong password (fail in confirm password).

Moreover, if user want to sign an existed username, system will warning the username is taken and open the forget password option for user (figure 10).

Also, alert will shows for no connection, no phone entered, and successfully registered (shows in figure 10,11,12) .

However, phone number is optional, the empty phone will only popup at the first time, user still can bind phone number in user profile interface.

Forget Password / Reset Password & Reset Phone Number

Back-end Support about Reset Function. Click Here

User who bind with phone can reset password. System will popup an alert to warn user that the target account to conform the account is correct after enter the phone number and new password (figure 13).

Backend will change the user’s password in database if user confirm this is the right account. An warning alert will popup if this phone number does not connect to any account (figure 14).

Also, as shown in Sign In Interface, empty phone number/password and two different password will be warning (part shows in figure 15).

The reset phone number interface is almost the same to reset password.

Main (Calculation) Interface

Back-end Support about Save Function. Click Here

As you can see in figure 16, there are many functionalities in Calculation Interface including reset display, add subject, calculate, save, and some input boxes fro input subject, grade, and weight.

Also, like Word or Excel, if you make a change but without save it to local or cloud, system will warn you before exit or logout (figure 17).

In addition subject part, user will get a warning if the marks or weights is not a number or empty subject (figure 18, 19). Meanwhile, the weights input box is a drop-write-box, default weight is 5, and you can choose 2.5, 5, 10, or any number as weight (figure 20).

The save function shows in figure 23 can achieve save to local (users’ pc) and cloud (database), but before save the final calculation must be made. Like if you calculate first, and then add another subject, the save function will popup a warning that need a final calculation (figure 21).

The saved information include time, title, grade and subject details. Time can be automatically be recorded using the pc’s time, but title need to be recorded by user (name the save).

User Profile

Back-end Support about User Profile. Click Here

In user profile, user can reset/bind phone number and reset password. Also, the latest record saved will be shown directly in the interface (figure 24). There are two more functions here: import and show all.

Import will import the latest record to the main interface (calculator), this functionality is used for calculate a full academic years grade without need to enter semester 1’s grade again.

Show all function (figure 25) can extent the windows of user profile, add a list in the right side, which is all the record extract from database. User can choose and click all the records. The record will be load to user profile main interface after click, and user can then import any record to Calculator.

Back-end Part

Back-end part is written in Django with over 15 APIs to achieve all the necessary between the front-end and database.

Authentication Center

All the sign in, log in, reset functions are achieved here in Authentication Center. Moreover, Http protocol used to connect to the APIs and use Json type in transmission.

  1. Log In:
    Front-end will send username and password get from the input box via a POST type. Using objects.filter() function to find if there exist the username. If find, then get the password and check if equal to the user input password…
    Go back to Log In Interface

  2. Sign In:
    The same as Log In, the back-end will receive username, password, phone number(optional), and an integer. The integer is used to count the times that no phone warning alert appears, the alert will only appear in the first time for just a remind usage.

    Then also use objects.filter() to check username if existed. If not, call an instance of the database and add a new row in the user database.

    At the same time, using dynamic_model = type(table_name, (models.Model,) to create a dynamic table called username_storage to store the record from calculator.

    Go back to Sign In Interface

  3. Reset:
    Use objects.get() to get the item the want to change and use row.save() to save the change.

    Go back to Reset Interface

RecordStorage

All functionalities related the records save and extract are achieved here. Http and Json also be employed.
