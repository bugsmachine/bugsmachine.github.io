CPT203 Software Engineering I 1: Introduction

CPT203 Software Engineering I 1: Introduction

Computer software is the product that software professionals build and then support over the long term. It encompasses computer programs and associated documentation that execute within a computer of any size and architecture .

Question1: Which of the following are the activities when you are developing a software?

  • A. Test the software
  • B. Write program
  • C. Estimate the time needed to complete the program
  • D. Run the program and let the user use it
  • E. Write a user guide
Click to view answer Answer: A,B,C,D,E,F (all)

Software Failure & Success

Software failures are a consequence of two factors:

  1. Increasing demands
  2. Low expectations (you don’t expect the software to grow into a giant beast)

Question2: Which of the following are considered to be the causes of failure in software engineering?

  • A. Changing software requirements
  • B. Adding software budget
  • C. Starting software development immediately to save time
Click to view answer Answer: A,C
×B. Adding a budget is not typically considered as a failure cause because having an adequate budget is often necessary for success! Not lead to failure!!

!!!Professional Software Development & Software Engineering Approach

Software engineering is intended to be professional team work, rather than individual programming. Usually, It includes techniques that support program specification, design, validation, and evolution. These activities are involved in systematic approach aka. software process.

Question3: Software engineering include the following activities__

  • A. Software team forming
  • B. Software specification
  • C. Software selection
  • D. Software validation
Click to view answer Answer: B,D
Four activities are involved:
1. Program specification 2. Program design 3. Program validation 4. Program evolution

!!!Four Important Attributes that All Professional Software Should Have

  1. Maintainability:
    Software should be written in such a way that it can involved to meet the changing needs if customers. This is critical because software change is an inevitable requirement to a changing business environment.
  2. Dependability and security:
    Dependability involves a range of characteristics including security, reliability and safety. There should not have physical or economic damage when a system failure happen to a dependable software. It should denied malicious users.
  3. Efficiency:
    Efficient software should not waste system resources such as memory and processor cycles. Efficiency should involves responsiveness, processing time, memory utilization…
  4. Acceptable:
    Software need to be acceptable to the target users. It need to be understandable, usable and compatible with other OS systems.

!!!Two Kinds of Software Products

  1. Generic software products: Sold on open market to any custom who is able to buy.
  2. Customized software products: Commission by a particular customer group and use by only a part of customers.

!!!Software Deterioration

Under ideal condition, due to software donot have ‘wear out’, the product’s failure rate will tend to be gentle with time goes after bugs or some problems fix (figure 1).

However, software still have deterioration. During its life, software will undergo many changes (for fix bugs, add functionalities…). As changes are made, the failure rate will spike again due to the side effect before it approach original steady-state failure rate (figure 2).

Solution to deterioration

  1. In order to reduce changes and steady the failure rate:
    • Work closely with stakeholder to ensure requirements are correctly defined
    • Improve requirement study approach to achieve better requirements study
  2. To reduce side effects after changes:
    • Modular software that changes to one part will not have a lot side effects on other parts
    • Maintainable!!!
    • Comprehensive testing!!!

!!!General Issues Affect Most Software

  1. Heterogeneity:
    Software system needs to be compatible to all OS systems…
  2. Business and social change:
    Business and social change are changing incredibly quickly as emerging economies develop and new technologies become available. Software need to keep up with these changes
  3. Security and trust
    It is needed to gain users trust.

!!!Web-based Software Engineering

  1. Software reuse is the dominant approach for constructing web-based systems (assemble from pre-existing software components and systems).

  2. Web-based systems should be developed and delivered incrementally. Because now people realize that it is nearly impractical to specify all requirements in advance.

  3. User interfaces are constrained by the capabilities of web browsers…

Question5: Why a web-based systems should be developed and delivered incrementally?

  • A. It is normally run on a web browser
  • B. The user is normally diverse, requirements can be unpredictable
  • C. To allow the developers to learn from the feedback as they are developing the system
  • D. The web is too large
Click to view answer Answer: C
It is nearly impractical to specify all requirements in advance. So, it is necessary to learn from the feedback as they are developing the system.

!!!Software Development Risk

  1. Development runs into difficulties for some reasons such as not work as expected, over budget and late delivery…
  2. Functionalities wasted such as wrong requirements, no target customers, cannot get users like…
  3. Wrong product: developers not understand or validate requirements
  4. Client satisfaction is the primary measurement of success in software project!!!

!!!How to Minimize Risk

Communication with the client:

  • Feasibility studies
  • Separation of requirements from designs
  • Validate software requirements
  • Milestones and releases
  • Acceptance and user testing
