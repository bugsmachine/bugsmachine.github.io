INT201 Decision Computation and Language 2: Deterministic Finite Automata (DFA)

INT201 Decision Computation and Language 2: Deterministic Finite Automata (DFA)

DFA is a finite-state machine that accepts or rejects a given string of symbols, by running through a state sequence uniquely determined by the string.

It is invented to recognize a special class of formal language and have many practical applications:

  • Lexical analysis -> scans the input program from beginning to end and divides it into tokens like identifiers, constants, and keywords and to remove comments and white space (specify tokens of programming languages)
  • Model checking, reasoning about systems with objective of proving they satisfy useful properties
  • Statistical models for analyzing biological and textual sequences

Introduction to DFA

Tuples of DFA

A DFA is defined as a 5-tuple:

M = (Q, Σ, δ, q, F)

Where

  1. Q is a finite set of states (In figure 1, q1, q2, q3 ∈ Q. State need to be surrounded with single circle),
  2. Σ is a finite set of symbols, called the alphabet of the automaton (In figure 1, Σ = {0,1}),
  3. δ : Q × Σ → Q is a function, called the transition function (δ will be detailed discuss later in next section of this blog),
  4. q ∈ Q is called the initial state (In figure 1, q1 is initial state),
  5. F ⊆ Q is a set of accepting/terminal states (In figure 1, q3 is terminal states, need to be surrounded with double circle).

Transition function δ

In example of figure 1, one of the transition function is q2 = 𝛿 (q1, 1). This indicate when at state q1, input ‘1’ will lead to jump to state q2.

Extend of δ: δ can be extend to δ*

Normal δ can only accept a letter as a input, like 𝛿 (q1, 1), 1 is only a letter. However, extended 𝛿 aka. 𝛿*, can accept a string (word).

δ*(q, ϵ) = q for all q ∈ Q
δ* (q, w𝑎) = δ (δ* (q, w) , 𝑎) for all q ∈ Q; w ∈ Σ* ; a ∈ Σ

Example:

δ (q0, 𝑎) = q1 and δ (q1, b) = q2 we can get -> δ* (q0, 𝑎b) = q2

Symbolic description of the example DFA

Automaton M = (Q, Σ, δ, q, F)
Set of states Q = {i, t, r}, Σ = {0, 1}, F = {t} and the transition function δ is given by

δ(i, 0) = r     δ(i, 1) = t
δ(t, 0) = t     δ(i, 1) = t
δ(r, 0) = r     δ(r, 1) = r

If δ is a partial function (not defined for some state/letter pairs shows in figure 3), then the DFA rejects an input if it ever encounters such a pair.

Language defined by DFA

Suppose we have a DFA M. A word w ∈ Σ* is said to be accepted or recognized by M if δ*(q0, w) ∈ F, otherwise it is said to be rejected. The set of all words accepted by M is called the language accepted by M and will be denoted by L(M). Thus

L(M) = {w ∈ Σ* : δ*(q0, w) ∈ F}

Any finite language is accepted by some DFA
A language A is called regular, if there exists a finite automaton M such that A = L(M) (This means A is regular if a can be recognized by a DFA)

Regular operations on languages

Let A and B be two languages over the same alphabet.
The union of A and B is defined as:

A ∪ B = {w : w ∈ A or w ∈ B}

The concatenation of A and B is defined as:

AB = {ww′ : w ∈ A and w′ ∈ B}

The star of A is defined as:

A* = {u1 u2...uk: k ≥ 0 and ui ∈ A for all i = 1, 2, . . . , k}

Example of A:
Given two languages A = {0, 01} and B = {1, 10}. Then

A ∪ B = {0, 01, 1, 10}
AB = {01, 010, 011, 0110}
A* = {ϵ, 0, 01, 00, 001, 010, 0101, 000, 0001, 00101, . . .}

Theorem

The set of regular languages is closed under the union operation, i.e., if A and B are regular languages over the same alphabet Σ, then A ∪ B is also a regular language.
